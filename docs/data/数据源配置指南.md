# 数据源配置指南

本文说明项目中不同数据源的配置方式、配置生效顺序、验证方法和常见错误。当前仓库默认配置为：

- 主行情读取源：`data_provider.source = "tdx"`
- 历史同步写入库：`history_sync.write_mode = "direct_db"`，`history_sync.direct_db_source = "duckdb"`
- 股票池刷新来源：`history_sync.stock_list_provider = "auto"`

这三个配置分别控制不同环节，不要混用：

- `data_provider.source`：回测、实盘监控、历史同步读取行情时使用的主数据源。
- `history_sync.direct_db_source`：历史同步选择 `direct_db` 写入模式时，写入到哪个数据库。
- `history_sync.stock_list_provider`：刷新同步股票池 `data/stock_list.csv` 时从哪里拿股票列表。

## 配置生效顺序

基础配置在 `config.json`。密钥、密码、私有地址建议放到 `config.private.json`，或通过环境变量 `CONFIG_PRIVATE_PATH` 指向私有配置文件。

项目会先读取 `config.json`，再合并私有配置。默认允许从私有配置覆盖这些敏感字段：

- `data_provider.tushare_token`
- `data_provider.default_api_key`
- `data_provider.default_api_url`
- `data_provider.llm_api_key`
- `data_provider.strategy_llm_api_key`
- `data_provider.api_key`
- `data_provider.mysql_password`
- `data_provider.postgres_password`
- `webhook_notification.feishu_webhook_url`
- `webhook_notification.feishu_secret`
- `evolution.llm.api_key`

此外，私有配置中的 `targets` 和 `strategies.active_ids` 如果非空，也会透传合并，便于在本机维护私有标的池和启用策略列表。

如果需要调整可从私有配置覆盖的字段集合，可以设置环境变量：

```bash
export CONFIG_PRIVATE_OVERRIDE_PATHS=data_provider.tushare_token,data_provider.default_api_key,data_provider.mysql_password
```

实盘启动 `python run_live.py` 还有额外覆盖：

- `DATA_PROVIDER`：覆盖 `data_provider.source`
- `TUSHARE_TOKEN`：覆盖 `data_provider.tushare_token`

推荐做法：

```json
{
  "data_provider": {
    "source": "tdx",
    "tdxdir": "D:\\new_tdx64",
    "tdx_host": "",
    "tdx_port": 0,
    "tdx_node_list": ""
  }
}
```

私有配置示例：

```json
{
  "data_provider": {
    "tushare_token": "YOUR_TUSHARE_TOKEN",
    "default_api_url": "https://your-market-api.example.com",
    "default_api_key": "YOUR_API_KEY",
    "mysql_password": "YOUR_MYSQL_PASSWORD",
    "postgres_password": "YOUR_POSTGRES_PASSWORD"
  }
}
```

如果私有文件不在项目根目录：

```bash
export CONFIG_PRIVATE_PATH=/path/to/config.private.json
python server.py
```

## 主行情数据源

主行情数据源由 `data_provider.source` 决定，可选值包括：

- `tdx`
- `akshare`
- `tushare`
- `mysql`
- `postgresql`
- `duckdb`
- `default`

### TDX / 通达信

适合本地已有通达信数据，或希望使用 TDX 网络节点获取行情。当前项目默认就是 `tdx`。

本地目录模式：

```json
{
  "data_provider": {
    "source": "tdx",
    "tdxdir": "D:\\new_tdx64",
    "tdx_timeout_sec": 120,
    "tdx_retry_count": 0,
    "tdx_max_bars": 0
  }
}
```

要求：

- `tdxdir` 应指向通达信安装目录或数据目录的上级。
- 该目录下应能找到 `vipdoc`。
- Windows 本机通达信目录通常更稳定。

网络节点模式：

```json
{
  "data_provider": {
    "source": "tdx",
    "tdxdir": "",
    "tdx_host": "119.147.212.81",
    "tdx_port": 7709,
    "tdx_node_list": "119.147.212.81:7709,119.147.212.81:7727",
    "tdx_timeout_sec": 120
  }
}
```

说明：

- `tdxdir` 为空时，会走网络节点或内置默认节点。
- `tdx_host`/`tdx_port` 指定单个节点。
- `tdx_node_list` 可配置多个候选节点，用英文逗号分隔。
- 如果节点不可达，连通性检查会失败。

### AkShare

适合快速试用。无需 token，但依赖 AkShare 接口和网络环境。

```json
{
  "data_provider": {
    "source": "akshare"
  }
}
```

注意：

- 不需要密钥。
- 免费接口可能限流、变更或偶发不可用。
- 股票池刷新默认也会优先尝试 AkShare。

### TuShare

适合有 TuShare Pro token 的用户。

`config.json`：

```json
{
  "data_provider": {
    "source": "tushare",
    "tushare_api_url": "http://tushare.xyz"
  }
}
```

`config.private.json`：

```json
{
  "data_provider": {
    "tushare_token": "YOUR_TUSHARE_TOKEN"
  }
}
```

也可以临时通过环境变量启动实盘：

```bash
export DATA_PROVIDER=tushare
export TUSHARE_TOKEN=YOUR_TUSHARE_TOKEN
python run_live.py
```

注意：

- `tushare_token` 为空时，`run_live.py` 会把实盘数据源从 `tushare` 回退到 `akshare`。
- 配置中心连通性检查会明确提示 token 未配置或不可用。

### MySQL

适合已有 MySQL 行情库。项目会按表名读取 `code`、`trade_time`、`open`、`high`、`low`、`close`、`vol`、`amount`。

`config.json`：

```json
{
  "data_provider": {
    "source": "mysql",
    "mysql_host": "127.0.0.1",
    "mysql_port": 3306,
    "mysql_user": "root",
    "mysql_database": "quantifydata",
    "mysql_charset": "utf8mb4",
    "mysql_query_page_size": 20000,
    "mysql_table_1min": "dat_1mins",
    "mysql_table_5min": "dat_5mins",
    "mysql_table_10min": "dat_10mins",
    "mysql_table_15min": "dat_15mins",
    "mysql_table_30min": "dat_30mins",
    "mysql_table_60min": "dat_60mins",
    "mysql_table_day": "dat_day"
  }
}
```

`config.private.json`：

```json
{
  "data_provider": {
    "mysql_password": "YOUR_MYSQL_PASSWORD"
  }
}
```

要求：

- 账号需要查询权限。
- 如果用于历史同步写入，还需要插入或更新权限。
- 表结构可参考 `docs/data/历史数据源表结构.sql`。
- 表名只能使用字母、数字、下划线。

### PostgreSQL

适合已有 PostgreSQL 行情库。字段要求与 MySQL 基本一致。

`config.json`：

```json
{
  "data_provider": {
    "source": "postgresql",
    "postgres_host": "127.0.0.1",
    "postgres_port": 5432,
    "postgres_user": "quant",
    "postgres_database": "quantifydata",
    "postgres_schema": "public",
    "postgres_query_page_size": 20000,
    "postgres_table_1min": "dat_1mins",
    "postgres_table_5min": "dat_5mins",
    "postgres_table_10min": "dat_10mins",
    "postgres_table_15min": "dat_15mins",
    "postgres_table_30min": "dat_30mins",
    "postgres_table_60min": "dat_60mins",
    "postgres_table_day": "dat_day"
  }
}
```

`config.private.json`：

```json
{
  "data_provider": {
    "postgres_password": "YOUR_POSTGRES_PASSWORD"
  }
}
```

要求：

- `postgres_schema` 默认是 `public`。
- 表需要在对应 schema 下。
- 账号需要查询权限；用于历史同步写入时还需要写入权限。

### DuckDB

适合本地单文件行情库。当前默认历史同步写入目标是 DuckDB。

```json
{
  "data_provider": {
    "source": "duckdb",
    "duckdb_path": "G:/duckdb/quantifydata.duckdb",
    "duckdb_query_page_size": 20000,
    "duckdb_table_1min": "dat_1mins",
    "duckdb_table_5min": "dat_5mins",
    "duckdb_table_10min": "dat_10mins",
    "duckdb_table_15min": "dat_15mins",
    "duckdb_table_30min": "dat_30mins",
    "duckdb_table_60min": "dat_60mins",
    "duckdb_table_day": "dat_day"
  }
}
```

要求：

- `duckdb_path` 指向可读的 `.duckdb` 文件。
- 如果用于历史同步写入，文件所在目录需要可写。
- 表名需要与配置一致。

### Default API

`default` 是自备 HTTP 行情 API 模式。

`config.json`：

```json
{
  "data_provider": {
    "source": "default"
  }
}
```

`config.private.json`：

```json
{
  "data_provider": {
    "default_api_url": "https://your-market-api.example.com",
    "default_api_key": "YOUR_API_KEY"
  }
}
```

项目会尝试这些接口：

- `GET /market/minutes`：分钟 K 线。
- `GET /market/latest`：最新行情。
- `GET /market/daily-bars/range`：日线区间。
- `GET /tables/{table}/rows`：多周期表查询兜底。
- `POST /tables/{table}/rows`：历史同步 `write_mode=api` 时写入数据。

认证头会尝试：

- `X-API-Key: <key>`
- `Authorization: Bearer <key>`

返回数据需要能归一化出这些字段：

- `code`
- `dt` 或 `trade_time` / `time` / `datetime` / `timestamp`
- `open`
- `high`
- `low`
- `close`
- `vol`
- `amount`

响应体可以直接是数组，也可以把数组放在 `rows`、`data`、`items`、`list`、`result` 中。

## 历史同步写入配置

历史同步读取源仍然看 `data_provider.source`。写入目标由 `history_sync.write_mode` 和 `history_sync.direct_db_source` 决定。

当前默认：

```json
{
  "history_sync": {
    "write_mode": "direct_db",
    "direct_db_source": "duckdb",
    "stock_list_provider": "auto",
    "stock_list_output_path": "data/stock_list.csv"
  }
}
```

`write_mode` 可选：

- `direct_db`：直接写数据库。
- `api`：写入自备默认 API，需要配置 `data_provider.default_api_url` 和 `data_provider.default_api_key`。

`direct_db_source` 可选：

- `mysql`
- `postgresql`
- `duckdb`

典型组合：

```json
{
  "data_provider": {
    "source": "tdx",
    "duckdb_path": "G:/duckdb/quantifydata.duckdb"
  },
  "history_sync": {
    "write_mode": "direct_db",
    "direct_db_source": "duckdb"
  }
}
```

这个组合表示：从 TDX 读取行情，写入 DuckDB。

股票池刷新来源：

- `auto`：先 AkShare，失败后 TuShare。
- `akshare`：只用 AkShare。
- `tushare`：只用 TuShare，需要 `tushare_token`。

刷新命令：

```bash
python scripts/update_history_sync_stock_list.py --provider auto
```

指定输出路径：

```bash
python scripts/update_history_sync_stock_list.py --provider akshare --output data/stock_list.csv
```

## 表结构要求

数据库类数据源读取和写入统一使用以下核心字段：

- `code`：股票代码，例如 `000001.SZ`
- `trade_time`：交易时间
- `open`
- `high`
- `low`
- `close`
- `vol`
- `amount`

默认表名：

- `dat_1mins`
- `dat_5mins`
- `dat_10mins`
- `dat_15mins`
- `dat_30mins`
- `dat_60mins`
- `dat_day`

MySQL 建表示例见 `docs/data/历史数据源表结构.sql`。PostgreSQL 和 DuckDB 可沿用同名字段与唯一键思路，至少保证 `code + trade_time` 可用于去重或更新。

## 验证方法

### 配置中心检查

启动服务：

```bash
python server.py
```

打开前端配置中心，选择数据源后执行连通性检查。检查会按当前表单草稿构建 provider，因此不一定需要先保存才能测试。

### 回测启动前检查

回测启动时会做数据源 precheck，日志中会出现类似：

```text
回测启动前数据源检测：source=tdx code=600036.SH
数据源连通性检测通过：source=tdx
```

如果失败，会返回 `source` 和具体原因。

### 实盘启动检查

```bash
python run_live.py
```

启动日志会输出：

```text
Selected Data Provider: tdx
```

如果设置了环境变量：

```bash
DATA_PROVIDER=akshare python run_live.py
```

日志应显示 `Selected Data Provider: akshare`。

### 命令行快速回测

```bash
python run_backtest.py --stock 600036.SH --start 2024-01-01 --end 2024-03-01
```

该命令会读取 `data_provider.source`，并使用配置中的主数据源。

## 常见问题

### TDX 连通失败

检查：

- `data_provider.tdxdir` 是否指向正确目录。
- `tdxdir` 下是否存在 `vipdoc`。
- 网络节点模式下 `tdx_host`、`tdx_port` 或 `tdx_node_list` 是否可访问。
- `tdx_timeout_sec` 是否过短。

### TuShare token 未配置

检查：

- `config.private.json` 是否存在。
- `CONFIG_PRIVATE_PATH` 是否指向了正确文件。
- `data_provider.tushare_token` 是否为空。
- 实盘启动时是否需要设置 `TUSHARE_TOKEN`。

### MySQL 或 PostgreSQL 连接失败

检查：

- host、port、user、password、database 是否正确。
- 账号是否有查询权限。
- PostgreSQL 的 `postgres_schema` 是否正确。
- 服务是否允许当前机器访问。

### 表名或字段不匹配

检查：

- `*_table_1min` 和 `*_table_day` 是否对应真实表。
- 表名是否只包含字母、数字、下划线。
- 表内是否有 `code`、`trade_time`、`open`、`high`、`low`、`close`、`vol`、`amount`。
- 股票代码格式是否与库内一致，例如 `600036.SH` 或 `600036`。

### DuckDB 路径不可读或不可写

检查：

- `data_provider.duckdb_path` 是否指向真实文件。
- 当前用户是否有读取权限。
- 历史同步写入时，文件和目录是否可写。
- 是否有其他进程独占该 DuckDB 文件。

### Default API 返回列缺失

检查：

- `/market/minutes` 是否返回数组或包含数组的对象。
- 每条记录是否包含时间列、OHLC、成交量和成交额。
- 时间列名是否是 `dt`、`trade_time`、`time`、`datetime` 或 `timestamp` 之一。
- HTTP 认证头是否和服务端约定一致。

## 推荐配置组合

新手本地试用：

```json
{
  "data_provider": {
    "source": "akshare"
  }
}
```

本地通达信优先：

```json
{
  "data_provider": {
    "source": "tdx",
    "tdxdir": "D:\\new_tdx64"
  }
}
```

TDX 读取，DuckDB 落库：

```json
{
  "data_provider": {
    "source": "tdx",
    "duckdb_path": "G:/duckdb/quantifydata.duckdb"
  },
  "history_sync": {
    "write_mode": "direct_db",
    "direct_db_source": "duckdb"
  }
}
```

自建 MySQL 行情库：

```json
{
  "data_provider": {
    "source": "mysql",
    "mysql_host": "127.0.0.1",
    "mysql_port": 3306,
    "mysql_user": "root",
    "mysql_database": "quantifydata",
    "mysql_table_1min": "dat_1mins",
    "mysql_table_day": "dat_day"
  }
}
```

生产或长期使用时，优先把 token、密码、API key 放入私有配置文件，避免提交到 Git。
